================================================================================
                        1. INTRODUCTION AU PROJET
================================================================================

Le projet LSL (Sign Language) est une application web de traduction en temps
réel de la langue des signes américaine (ASL) en texte. Le système utilise
l'intelligence artificielle et l'apprentissage automatique pour reconnaître
les gestes des mains et les convertir en lettres de l'alphabet.

OBJECTIFS PRINCIPAUX:
- Traduction en temps réel des signes en texte
- Interface webcam intuitive avec suivi des mains
- Architecture microservices moderne
- Authentification et autorisation sécurisées
- Historique des traductions pour chaque utilisateur

PUBLIC CIBLE:
- Personnes sourdes ou malentendantes
- Interprètes en langue des signes
- Établissements éducatifs
- Projet académique de démonstration

5.1 CORRECTION DE LA PRÉDICTION INSTANTANÉE
=============================================

PROBLÈME:
Les prédictions passaient par RabbitMQ queue et retournaient "QUEUED" au lieu
du résultat réel. L'utilisateur ne voyait jamais la lettre prédite.

SOLUTION:
- Changé l'endpoint de `/api/predict/` (queue) vers `/ai/predict/` (direct)
- Mis à jour camera.js pour appeler directement le AI service
- Configuration du middleware Traefik StripPrefix

FICHIERS MODIFIÉS:
- frontend/static/js/camera.js
  * Ligne ~394: fetch('/ai/predict/') au lieu de fetch('/api/predict/')
  * Ligne ~627: apiUrl = '/ai/predict/'

- docker-compose.yml
  * Ajout middleware: traefik.http.routers.ai.middlewares=ai-strip-prefix
  * Configuration: traefik.http.middlewares.ai-strip-prefix.stripprefix.prefixes=/ai

RÉSULTAT:
✓ Prédictions instantanées (2-3 secondes)
✓ Affichage correct de la lettre prédite
✓ Score de confiance affiché
✓ Top 3 predictions disponibles

5.2 CONFIGURATION TRAEFIK ROUTING
===================================

PROBLÈME:
Traefik forwards les requêtes avec le préfixe `/ai` intact, causant des erreurs
404 car le AI service n'a que la route `/predict/`.

SOLUTION:
Ajout d'un middleware StripPrefix pour supprimer `/ai` avant le forwarding.

CONFIGURATION AJOUTÉE:
```yaml
labels:
  - "traefik.http.routers.ai.middlewares=ai-strip-prefix"
  - "traefik.http.middlewares.ai-strip-prefix.stripprefix.prefixes=/ai"
```

RÉSULTAT:
✓ Routing correct: `/ai/predict/` → `/predict/`
✓ Plus d'erreurs 404
✓ Communication fluide entre services

5.6 SAUVEGARDE HISTORIQUE DES PRÉDICTIONS
===========================================

PROBLÈME:
Les prédictions étaient affichées mais jamais sauvegardées dans l'historique
de l'utilisateur.

SOLUTION:
- Création fonction saveToHistory() dans camera.js
- Appel automatique après chaque prédiction
- Correction de l'API endpoint History POST
- Filtrage des prédictions incertaines

FICHIERS MODIFIÉS:
- frontend/static/js/camera.js
  * Fonction saveToHistory() créée (~605-647)
  * Appel dans captureImage() après affichage
  * Appel dans predictAlphabetFromHand() (auto-predict)
  
- api_service/core/views.py
  * HistoryView.post() corrigé
  * Accepte translation (texte) et confidence (float)
  * Au lieu de translation_id (ForeignKey)

- api_service/api_service/settings.py
  * ALLOWED_HOSTS inclut maintenant 'lsl_django'

LOGIQUE DE SAUVETAGE:
```javascript
function saveToHistory(text, confidence) {
  // Ne sauvegarde pas les prédictions incertaines
  if (text === '?' || text === 'QUEUED' || !text) {
    return; // Skip
  }
  
  // Sauvegarde via API
  fetch('/api/history/', {
    method: 'POST',
    body: JSON.stringify({
      translation: text,
      confidence: confidence
    })
  })
}
```

RÉSULTAT:
✓ Chaque prédiction sauvegardée automatiquement
✓ Historique accessible par utilisateur
✓ Score de confiance conservé
✓ Interface history fonctionnelle
